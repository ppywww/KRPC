# Krpc 项目敏捷开发文档

## 1. 项目概述

### 1.1 项目基本信息

| 属性 | 说明 |
|------|------|
| 项目名称 | Krpc |
| 项目类型 | C++ RPC 分布式框架 |
| 核心技术栈 | Muduo、Protobuf、ZooKeeper、glog |
| C++ 标准 | C++11 |
| 目标用户 | 需要构建分布式微服务系统的开发团队 |

### 1.2 项目目标

- 实现高性能 RPC 通信框架
- 支持服务注册与发现机制
- 提供自定义协议解决 TCP 粘包/拆包问题
- 基于 Muduo 库实现 Reactor 网络模型

## 4. 敏捷 ceremonies (仪式)

### 4.1 Sprint 计划会议 (Sprint Planning)

- **参与人**: 全体团队成员
- **时长**: 2 小时
- **议程**:
  1. 产品负责人介绍 Sprint 目标
  2. 团队讨论需求细节
  3. 团队估算工作量
  4. 团队承诺 Sprint 待办列表

### 4.2 每日站会 (Daily Standup)

- **参与人**: 全体团队成员
- **时长**: 15 分钟
- **问题模板**:
  - 昨天完成了什么？
  - 今天计划做什么？
  - 遇到什么障碍？

### 4.3 Sprint 评审会议 (Sprint Review)

- **参与人**: 团队 + 利益相关者
- **时长**: 1.5 小时
- **议程**:
  1. 演示已完成的功能
  2. 收集反馈
  3. 更新产品待办列表

### 4.4 Sprint 回顾会议 (Sprint Retrospective)

- **参与人**: 全体团队成员
- **时长**: 1 小时
- **议程**:
  1. 回顾 Sprint 执行情况
  2. 识别改进点
  3. 制定改进计划

## 6. 代码协作规范

### 6.1 Git 工作流程

采用 **Git Flow** 分支模型:

```
main (生产分支)
  └── develop (开发分支)
        ├── feature/xxx (功能分支)
        ├── release/xxx (发布分支)
        └── hotfix/xxx (热修复分支)
```

### 6.2 分支命名规范

| 分支类型 | 命名格式 | 示例 |
|----------|----------|------|
| 功能分支 | feature/功能描述 | feature/load-balance |
| 发布分支 | release/v版本号 | release/v1.0.0 |
| 热修复分支 | hotfix/问题描述 | hotfix/fix-connection-leak |
| 开发分支 | develop | develop |

### 6.3 提交信息规范

```
<类型>(<模块>): <简短描述>

类型: feat | fix | docs | style | refactor | test | chore

示例:
feat(provider): 添加服务注册健康检查功能
fix(channel): 修复连接关闭时资源泄漏问题
docs(readme): 更新项目文档
```

### 6.4 Code Review 清单

- [ ] 代码逻辑正确性
- [ ] 是否符合编码规范
- [ ] 是否有潜在的 bug
- [ ] 单元测试覆盖率
- [ ] 性能影响评估
- [ ] 安全性检查

## 7. 技术规范

### 7.1 核心模块编码规范

| 模块 | 头文件 | 实现文件 | 依赖 |
|------|--------|----------|------|
| 应用层 | include/Krpc*.h | Krpc*.cc | 配置、日志 |
| 网络层 | include/Krpcchannel.h | Krpcchannel.cc | Protobuf、Muduo |
| 服务层 | include/Krpcprovider.h | Krpcprovider.cc | Muduo |
| 注册中心 | include/zookeeperutil.h | zookeeperutil.cc | ZooKeeper |

### 7.2 协议设计规范

```
RPC 消息格式:
[4B Total Length][4B Header Length][Header Data][Body Data]

Header (Protobuf):
- service_name: bytes
- method_name: bytes
- args_size: uint32
```

### 7.3 错误处理规范

```cpp
// 使用 Krpccontroller 记录错误
controller->SetFailed("错误描述");

// 使用 KrpcLogger 记录日志
LOG(ERROR) << "错误信息";
```
